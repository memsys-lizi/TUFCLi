# TUF API 关卡接口详细文档

## 1. 关卡搜索接口

### 端点
```
GET /v2/database/levels
```

### 描述
使用过滤器搜索关卡（query、pguRange、sort、tags 等）。支持分页。

### 查询参数

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| `query` | string | 否 | 搜索文本，可以搜索关卡名称、歌曲名、艺术家等 |
| `facetQuery` | string | 否 | Facet 过滤器 JSON v1（tags + curationTypes + specialDifficulties）<br/>用于替代 tagsFilter 和 curatedTypesFilter<br/>详见 4.1 节 |
| `pguRange` | string | 否 | PGU（难度）范围过滤，格式示例：`"0-100"`<br/>支持难度滑块选择，详见 4.2 节 |
| `sort` | string | 否 | 排序方式<br/>可选值：`id`, `passRecords`, `totalClears`, `favorites`, `baseScore`, `bpm`, `tileCount`, `duration`, `random`<br/>详见 4.3 节 |
| `order` | string | 否 | 排序方向：`asc`（升序）或 `desc`（降序，默认） |
| `page` | integer | 否 | 页码（基于页码的分页） |
| `offset` | integer | 否 | 偏移量（基于偏移量的分页） |
| `limit` | integer | 否 | 每页返回的结果数量限制（默认可能是 20 或 50） |
| `byCreatorId` | string | 否 | 按创作者 ID 过滤，只返回该创作者参与的关卡 |

### 请求示例

**示例 1: 基础搜索**
```http
GET /v2/database/levels?query=freedom&pguRange=50-100&sort=totalClears&order=desc&limit=20&offset=0
Authorization: Bearer YOUR_TOKEN
```

**示例 2: 使用标签过滤（Tech + Key Count）**
```http
GET /v2/database/levels?facetQuery=%7B%22tags%22%3A%5B%22Tech%22%2C%22Key%20Count%22%5D%7D&sort=favorites&limit=50
Authorization: Bearer YOUR_TOKEN
```

**示例 3: 特殊难度过滤（Unranked 关卡）**
```http
GET /v2/database/levels?facetQuery=%7B%22specialDifficulties%22%3A%5B%22Unranked%22%5D%7D&sort=id
Authorization: Bearer YOUR_TOKEN
```

**示例 4: 复合查询（谱面类型 + VFX + 难度范围）**
```http
GET /v2/database/levels?facetQuery=%7B%22curationTypes%22%3A%5B%22C3%22%2C%22V3%22%5D%7D&pguRange=100-150&sort=bpm&order=asc
Authorization: Bearer YOUR_TOKEN
```

## 2. 关卡详情接口

### 端点 1: 按 ID/Slug 获取
```
GET /v2/database/levels/{id}
```

### 端点 2: 按数字 ID 获取
```
GET /v2/database/levels/byId/{id}
```

### 描述
获取单个关卡的完整详情，包括元数据、评分、创作者信息等。结果会被缓存。

### 路径参数
- `id`: 关卡 ID（可以是数字 ID 或 slug）

### 请求示例
```http
GET /v2/database/levels/12345
Authorization: Bearer YOUR_TOKEN
```

### 响应字段（常见字段）
```json
{
  "id": 12345,
  "song": "歌曲名称",
  "artist": "艺术家名称",
  "creator": "创作者信息",
  "difficulty": 85.5,
  "bpm": 180,
  "tiles": 2500,
  "levelLengthInMs": 180000,
  "videoLink": "https://...",
  "dlLink": "下载链接",
  "workshopLink": "Steam Workshop 链接",
  "rating": {
    "difficulty": 8.5,
    "confirmedAt": "2024-01-01T00:00:00Z"
  },
  "tags": ["Tech", "Flow"],
  "downloadCount": 1000,
  "likeCount": 150,
  "hidden": false,
  "deleted": false,
  "createdAt": "2024-01-01T00:00:00Z",
  "updatedAt": "2024-01-02T00:00:00Z"
}
```

### 字段说明
- `id`: 关卡唯一标识符
- `song`: 歌曲名称
- `artist`: 艺术家/音乐制作者
- `creator`: 关卡创作者信息（可能是对象，包含 id、name 等）
- `difficulty` / `rating.difficulty`: 难度评分（PGU 值）
- `bpm`: 每分钟节拍数
- `tiles` / `tilecount`: 关卡瓦片数量
- `levelLengthInMs`: 关卡长度（毫秒）
- `videoLink`: 视频链接（YouTube、Bilibili 等）
- `dlLink`: 下载链接（外部链接）
- `workshopLink`: Steam Workshop 链接
- `rating`: 评分详情对象
- `tags`: 标签数组（如 Tech、Flow、Gimmick 等）
- `downloadCount`: 下载次数
- `likeCount`: 点赞数
- `hidden`: 是否隐藏
- `deleted`: 是否已删除
- `confirmedAt`: 评分确认时间

## 3. 关卡下载接口

### 3.1 获取关卡 CDN 数据
```
GET /v2/database/levels/{id}/cdnData
```

**描述**: 返回关卡的 CDN 相关数据，包括文件元数据、下载信息等。

**请求示例**:
```http
GET /v2/database/levels/12345/cdnData
Authorization: Bearer YOUR_TOKEN
```

**响应示例**:
```json
{
  "fileId": "uuid-string",
  "originalFilename": "level.zip",
  "size": 15728640,
  "metadata": {
    "targetLevelRelativePath": "level.adofai",
    "levelFiles": [...],
    "songFiles": [...]
  }
}
```

### 3.2 获取 .adofai 文件
```
GET /v2/database/levels/{id}/level.adofai
```

**描述**: 直接返回关卡的 .adofai 配置文件内容或重定向到 CDN 链接。

**请求示例**:
```http
GET /v2/database/levels/12345/level.adofai
Authorization: Bearer YOUR_TOKEN
```

### 3.3 Pack 下载链接生成
```
POST /v2/database/levels/packs/{id}/download-link
```

**描述**: 为 pack（关卡包）或文件夹生成 CDN 下载链接（zip 格式）。

**请求体**:
```json
{
  "folderId": 123,
  "downloadId": "optional-string",
  "trimFolderNames": true
}
```

**响应示例**:
```json
{
  "downloadLink": "https://cdn.tuforums.com/...",
  "expiresAt": "2024-01-01T12:00:00Z"
}
```

### 3.4 获取 Pack CDN 元数据
```
GET /v2/database/levels/packs/{id}/cdnData
```

**描述**: 获取 pack 中每个关卡的 CDN zip 大小、原始文件名和裁剪后的 LEVELZIP 元数据。

**响应字段**:
- `targetLevelRelativePath`: 目标关卡相对路径
- `levelFiles`: 关卡文件数组（name、relativePath、songFilename、size）
- `songFiles`: 歌曲文件数组（name、size）

## 4. 关卡搜索的高级过滤

### 4.1 facetQuery 参数格式

`facetQuery` 是一个 JSON 字符串，用于精确控制标签和策展类型的过滤：

```json
{
  "tags": ["Tech", "Flow"],
  "curationTypes": ["C3", "V2"]
}
```

**使用示例**:
```http
GET /v2/database/levels?facetQuery=%7B%22tags%22%3A%5B%22Tech%22%5D%7D
```

### 4.1.1 完整的标签分类（基于前端界面）

#### Chart（谱面类型）
- `C3`, `C2`, `C1`, `C0`

#### Ornamental（装饰性）
- `O3`, `O2`, `O1`

#### VFX（视觉效果）
- `V3`, `V2`, `V1`, `V0`

#### Misc（杂项）
- `H2`, `H1`, `Epic`

#### Play Style（游玩风格）
- `Pseudo` - 伪押
- `Rolling` - 滚键
- `Indexing` - 指法
- `Tech` - 技术向
- `Key Count` - 按键计数
- `Key Count+` - 按键计数增强
- `Feetdex` - 脚步指法（原文 Feetdex，非 Feedex）
- `Feet Switch` - 脚步切换

#### Key Limit（按键限制）
- `1 Key Limit` - 单键限制
- `2 Key Limit` - 双键限制
- `4 Key Limit` - 四键限制
- `6 Key Limit` - 六键限制
- `8 Key Limit` - 八键限制
- `10 Key Limit` - 十键限制
- `12 Key Limit` - 十二键限制
- `16 Key Limit` - 十六键限制
- `Overlay Allowed` - 允许叠键
- `2-Hand Pseudos` - 双手伪押
- `Onhand/Offhand Limit` - 主手/副手限制
- `Variable Key Limit` - 可变按键限制
- `No Large Keys` - 无大键
- `Feet Only` - 仅脚步

#### Judgement（判定机制）
- `Judgement Limit` - 判定限制
- `HP Bar` - 生命条
- `Detailed Judgement` - 详细判定
- `Timing Window Scale` - 时间窗口缩放

#### Gimmick（特殊机制）
- `Free Roam` - 自由漫游
- `Multi Track` - 多轨道
- `Math` - 数学
- `RPG` - RPG 元素

#### Other（其他）
- `Pure Perfect Basescore Increase` - 纯 Perfect 基础分增加

#### Gimmick（特殊机制）- 补充
- `Free Roam` - 自由漫游
- `Multi Track` - 多轨道
- `Math` - 数学
- `RPG` - RPG 元素
- `Memorization` - 记忆向
- `Unorthodox Reading` - 非正统读谱
- `Arrow Key` - 方向键
- `Camera Required` - 需要摄像机

#### VFX（视觉效果）- 标签补充
- `Full VFX` - 完整视效
- `Camera` - 摄像机效果
- `Filters` - 滤镜
- `Non-VFX` - 无视效
- `Decorations` - 装饰
- `Low VFX` - 低视效

#### Required Mods（必需模组）
- `Youtube Stream` - YouTube 串流
- `Key Limiter` - 按键限制器

#### DLC（可下载内容）
- `DLC` - DLC 内容
- `Hold` - 长按机制
- `Multi Planet` - 多星球

#### Marathon（马拉松/超长关卡）
- `30+ Minutes` - 30 分钟以上
- `45+ Minutes` - 45 分钟以上
- `1+ Hours` - 1 小时以上
- `1.5+ Hours` - 1.5 小时以上
- `2+ Hours` - 2 小时以上
- `Timeless` - 无时间限制

#### Misc（杂项）- 补充
- `Pure Perfect Basescore Edit` - 纯 Perfect 基础分编辑
- `Auto Tile` - 自动瓦片
- `Basescore Edit` - 基础分编辑

### 4.2 pguRange 参数（难度范围）

PGU（Player Gauge Units）是难度的度量单位。格式示例：
- `"0-50"`: 简单关卡
- `"50-100"`: 中等难度
- `"100-150"`: 困难关卡
- `"150+"`: 极难关卡

**难度等级标识**（从截图中的滑块）：
- `P1` - 最低难度起点
- `U2` - 超高难度 (U = Ultra?)
- `U3` - 极限难度

**特殊难度标签**：
- `Unranked` - 未评级
- `P0` - 零级（可能是教程级别）
- `Censored` - 已审查/屏蔽
- `Impossible` - 不可能完成

### 4.2.1 特殊难度过滤

在搜索时可以通过特殊参数过滤特定难度类型的关卡。这些选项可能通过独立的查询参数或 `facetQuery` 实现：

```json
{
  "specialDifficulties": ["Unranked", "Censored", "Impossible"]
}
```

### 4.3 sort 参数可用值（基于前端实现）

根据前端界面，以下是确认的排序选项：

| 排序字段 | 说明 |
|---------|------|
| `id` | 按 ID 排序（默认） |
| `passRecords` | 按烟关记录排序 |
| `totalClears` | 按总通关数排序 |
| `favorites` | 按收藏数排序 |
| `baseScore` | 按基础分排序 |
| `bpm` | 按 BPM（节奏）排序 |
| `tileCount` | 按物块数排序 |
| `duration` | 按时长排序 |
| `random` | 随机排序 |

**排序方向**：配合 `order` 参数使用
- `asc`: 升序
- `desc`: 降序（默认）

## 5. 其他关卡相关接口

### 5.1 检查关卡是否被点赞
```
GET /v2/database/levels/{id}/isLiked
```

### 5.2 点赞/取消点赞关卡
```
PUT /v2/database/levels/{id}/like
```

```json
{
```

```
GET /v2/database/levels/{id}/ratings
```

### 5.4 获取关卡别名
```
GET /v2/database/levels/{id}/aliases
```

**描述**: 返回歌曲/艺术家的所有别名（已废弃）。

### 5.5 检查关卡访问权限
```
HEAD /v2/database/levels/{id}
```

**描述**: 返回 200 表示关卡存在且用户有访问权限，403/404 表示无权限或不存在。

## 6. 认证说明

所有关卡接口都需要 Bearer Token 认证：

```http
Authorization: Bearer YOUR_ACCESS_TOKEN
```

获取 Token 的方式：
1. 登录接口: `POST /v2/auth/login`
2. OAuth 回调: `POST /v2/auth/oauth/callback/{provider}`


---

## 7. 错误响应

所有接口的错误响应格式统一：

```json
{
  "error": "错误类型",
  "message": "详细错误信息"
}
```

常见错误代码：
- `400`: 请求参数错误
- `401`: 未认证
- `403`: 无权限访问
- `404`: 资源不存在
- `500`: 服务器内部错误


---

## 8. 缓存策略

- 关卡搜索结果会被缓存
- 关卡详情会被缓存（Redis）
- CDN 元数据有独立的缓存层
- Pack 树结构使用分层缓存（结构 vs ES 关卡负载）

## 11. 前端筛选器详细说明

### 11.1 筛选设置面板结构

根据截图，前端筛选器分为四个主要部分：

1. **特殊难度** (Special Difficulty)
   - Unranked, P0, Censored, Impossible

2. **精选关卡类型** (Featured Level Types)
   - Chart: C3, C2, C1, C0
   - Ornamental: O3, O2, O1
   - VFX: V3, V2, V1, V0
   - Misc: H2, H1, Epic

3. **标签** (Tags) - 分为多个子类别
   - 其他 (Other)
   - Play Style（游玩风格）
   - Key Limit（按键限制）
   - Judgement（判定）
   - Gimmick（机制）

4. **眼睛图标按钮** - 可能用于切换可见性过滤

### 11.2 筛选器交互逻辑

- **单选 vs 多选**：从界面看，标签支持多选（可以同时选择多个标签）
- **全选按钮**：每个类别右侧有"全部切换"按钮
- **累积过滤**：多个筛选条件是 AND 逻辑关系（同时满足）
- **标签内 OR 逻辑**：同一类别内的多个标签可能是 OR 关系

## 12. 注意事项和最佳实践

### 12.1 性能优化建议

1. **合理设置 limit**：不要一次请求过多数据，建议 20-50 条
2. **使用缓存**：相同的查询条件会命中缓存，响应更快
3. **避免过于复杂的 facetQuery**：太多标签组合可能影响性能
4. **优先使用 offset 分页**：比 page 参数更灵活

### 12.2 常见错误

1. **facetQuery 格式错误**：必须是有效的 JSON 字符串且需要 URL 编码
2. **pguRange 格式错误**：应该是 `"min-max"` 格式，如 `"50-100"`
3. **sort 参数拼写错误**：参考第 4.3 节的确切字段名
4. **认证 Token 过期**：需要刷新或重新登录

1. 使用浏览器开发者工具查看前端实际发送的请求
2. 检查响应的 total 字段确认筛选是否生效
3. 逐步添加筛选条件，定位问题参数
